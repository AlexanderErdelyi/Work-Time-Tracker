# Documentation Contribution Guide

## Writing rules

- Use clear, task-focused language
- Prefer step-by-step instructions
- Keep screenshots current
- Avoid environment-specific assumptions

## Change workflow

1. Update source docs in `docs/`
2. Run `npm run sync-docs` from `Timekeeper.Web/`
3. Verify in-app rendering and links
4. Include docs updates in PR summary

## Review checklist

- Accurate feature names
- No broken links
- No outdated UI references
- Grammar and terminology consistency
